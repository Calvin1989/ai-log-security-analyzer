# 演示指南 (Demo Guide)

本指南旨在帮助您在 5-8 分钟内向面试官或团队展示 **AI Log Security Analyzer** 的完整功能。

---

## 准备阶段
- 启动环境：`docker compose up --build`
- 准备文件：`samples/nginx_access_sample.log`、`samples/apache_access_sample.log`，以及任意第 3 个可复用的 Web 访问日志样本

---

## 演示脚本 (5-8 分钟)

### 1. 快速启动与隐私优势 (1 分钟)
- **展示点**: Docker 启动后的简洁界面。
- **解说**: “项目基于前后端分离架构，核心逻辑完全本地运行。没有数据库，不连接外部 API，确保敏感的服务器日志绝不出本地。”

### 2. 单文件分析与解析健壮性 (1 分钟)
- **动作**: 切换顶部栏的 **"中文 | English"** 开关。
- **动作**: 上传 `nginx_access_sample.log`。
- **解说**: “支持中英文实时切换。解析器具备自动识别格式的能力，并能捕捉解析失败的样本行（在‘解析质量’卡片中查看），这在处理非标准日志时非常有用。”

### 3. 管理层摘要与风险发现 (1 分钟)
- **展示点**: 顶部的 **Executive Summary** 模块。
- **解说**: “系统会自动生成确定性的安全摘要和风险评分。这完全由本地规则驱动，不依赖 LLM，保证了分析结论的可复现性。”

### 4. 批量案件分析 (1 分钟)
- **动作**: 重新选择 2-3 个日志文件，演示多文件上传。
- **展示点**: 页面仍返回同一个案例级结果，但解析质量卡片增加 **Source Files** 明细表。
- **解说**: “v2.0 支持把多个日志文件作为一个 case 联合分析。后端会分别解析每个 source file，再把成功解析的日志汇总成一个统一案件视角。”

### 5. 攻击时间轴还原 (1 分钟)
- **展示点**: **Attack Timeline** 视图。
- **动作**: 过滤 Severity 为 **"HIGH"**，展开一个事件的 **"Evidence"**。
- **解说**: “时间轴按时间线还原攻击者行为，方便分析师快速理清攻击脉络。”

### 6. 检测透明度与解释 (1 分钟)
- **展示点**: **Rule Coverage & Detection Explainability** 面板。
- **解说**: “我们拒绝‘黑盒’。在这里可以看到系统应用的所有规则，哪些被触发了，以及具体的触发原因和脱敏后的证据样例。”

### 7. 交互式规则调优 (1 分钟)
- **动作**: 打开 **Temporary Rule Tuning** 面板。
- **动作**: 调整 “高频访问阈值” (例如从 10 改为 5)，点击 **"应用调优并重新分析"**。
- **展示点**: 观察 **Findings** 和 **Rule Coverage** 的实时变化。
- **展示点**: 如果当前是 batch 分析，指出面板中的 batch hint，说明调优将作用于整个批量集合。
- **解说**: “支持实时调优。分析师可以临时调整规则阈值或禁用特定规则，观察其对结果的影响，而无需修改任何配置文件。在 batch 模式下，这些规则会应用到整个案件集合。”

### 8. 本地历史与趋势分析 (1 分钟)
- **动作**: 再次上传同一个文件或另一个样本。
- **展示点**: 在 **Recent Analyses** 中查看刚才的批量记录，确认它显示了 **Batch** 标签。
- **动作**: 在 **Report Comparison** 中选择两条记录进行对比。
- **解说**: “本地历史会保留最近分析结果，并用 Batch 标签区分案件级分析。这样既能追踪历史，也能快速切换到对比视图。”

### 9. 合规分享与导出 (1 分钟)
- **动作**: 滚动到页面底部，点击 **"下载 Markdown 报告 (.md)"**，再按需演示 **"下载脱敏报告 (.md)"**。
- **解说**: “报告仍然完全由本地规则引擎生成。无论单文件还是批量案件，都可以导出 Markdown 报告用于复盘、归档或对外分享；如需外部沟通，还可以使用脱敏版本平衡深度分析与隐私保护。”

---

## 关键技术总结 (Takeaways)
- **Local-first**: 隐私优先。
- **Case-level Workflow**: 支持从单文件分析升级到多文件案件分析。
- **Explainability**: 规则透明，证据确凿。
- **Zero-Infra**: 零基础设施，启动即用。
- **Bilingual**: 友好的国际化支持。
